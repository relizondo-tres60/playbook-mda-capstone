🎯 Propósito del Procedimiento

Diagnosticar y resolver incidentes de hardware y software de base en laptops y desktops corporativos, incluyendo diagnóstico físico, formateo, mantenimiento preventivo y coordinación de reparación o reposición.

📋 Alcance

✓ Incluye

  • Diagnóstico de fallas de hardware (disco, RAM, batería, pantalla)
  • Formateo y reinstalación del sistema operativo
  • Mantenimiento preventivo (limpieza, actualizaciones)
  • Diagnóstico de lentitud extrema del equipo
  • Problemas de configuración del equipo a nivel de SO

✗ No incluye

  • Fallas de software de aplicación (ver SOP-APP-005)
  • Reposición de equipo (ver SOP-EQU-005)
  • Equipos de terceros o proveedores (no son activos Capstone)
✅ Requisitos Previos
  • Acceso al equipo (presencial o remoto)
  • Número de serie o nombre del equipo para registro
📄 Procedimiento Paso a Paso
Paso 01
Diagnóstico inicial del incidente

Conectarse remotamente al equipo o atender en sitio. Diagnóstico básico: verificar en Administrador de Tareas el uso de CPU, RAM y disco. Revisar Visor de Eventos para errores críticos. Para problemas de hardware físico (pantalla, teclado, etc.): atención presencial obligatoria.

⚠ Nota: En mina (MVE/MBL), la atención presencial requiere coordinar con el supervisor de turno para el acceso al equipo en campo.
Paso 02
Ejecutar diagnóstico de hardware si aplica

Para sospecha de falla de disco: ejecutar chkdsk y/o CrystalDiskInfo para verificar la salud del disco. Para sospecha de RAM: ejecutar Windows Memory Diagnostic. Para batería de laptop: revisar el reporte de estado de batería (powercfg /batteryreport). Documentar los resultados del diagnóstico.

⚠ Nota: Un disco con sectores defectuosos o con health < 80% en CrystalDiskInfo debe reemplazarse aunque 'aún funcione'. Prevenir es más barato que recuperar datos.
Paso 03
Ejecutar la solución correspondiente

Según el diagnóstico: (a) Disco dañado leve: chkdsk /f /r para reparar sectores. (b) Lentitud por software: limpiar programas de inicio, eliminar archivos temporales, desactivar aplicaciones innecesarias. (c) Problema de SO: formateo y reinstalación de imagen corporativa. (d) Hardware dañado: registrar para reposición (SOP-EQU-005).

⚠ Nota: Antes de formatear, asegurar backup de datos del usuario: verificar que OneDrive está sincronizado. Si no hay backup, coordinar con el usuario antes de proceder.
Paso 04
Post-configuración después de formateo

Si se formateó el equipo: unir al dominio AD, instalar agente de antivirus, aplicar políticas GPO, instalar software base (Office, Adobe Reader, cliente VPN). Restaurar datos del usuario desde OneDrive. Verificar que el usuario puede iniciar sesión con sus credenciales corporativas.

⚠ Nota: El proceso completo de formateo y configuración toma entre 2-4 horas. Coordinar con el usuario para asegurar disponibilidad del equipo.
Paso 05
Verificar y cerrar

Confirmar con el usuario que el equipo funciona correctamente. Si hubo diagnóstico de hardware grave (disco, RAM), documentar la recomendación de reposición en el ticket. Cerrar con 'Resolved' o derivar a SOP-EQU-005.

📡 Criterios de Escalamiento
  • Falla de hardware confirmada que requiere reposición → SOP-EQU-005
  • Problema de imagen corporativa o MDM (Intune) → N3 Server_Support
  • Equipo con datos sensibles que debe ser borrado de forma segura → N3 + protocolo de borrado seguro
