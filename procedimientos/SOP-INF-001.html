🎯 Propósito del Procedimiento

Gestionar la disponibilidad del sistema PI Vision y PI System utilizado para el monitoreo en tiempo real de procesos de planta (Desaladora, Planta MVE/MBL), incluyendo respuesta a caídas del servidor y coordinación con el especialista Pinnovartic.

📋 Alcance

✓ Incluye

  • Monitoreo de disponibilidad del servidor PI Vision
  • Reinicio del servidor OPC PI Vision ante caída
  • Activación del Protocolo P1 para fallas críticas
  • Gestión de accesos al servidor PI Vision
  • Coordinación con especialista Pinnovartic

✗ No incluye

  • Configuración de tags PI o integración con SCADA (especialista Pinnovartic)
  • Problemas de la plataforma PI de OSIsoft (soporte del proveedor)
✅ Requisitos Previos
  • Acceso de administración al servidor PI Vision
  • Contacto de Pinnovartic para soporte especializado
  • Conocimiento del protocolo P1 de incidentes críticos
📄 Procedimiento Paso a Paso
Paso 01
Recibir alerta y evaluar severidad

Al recibir alerta de PI Vision: verificar en el sistema de monitoreo si el servidor PI Vision está respondiendo. Verificar si el reporte de planta está afectado. Si el reporte de planta está caído: activar PROTOCOLO P1 de inmediato (ver paso P1). Si el servidor responde pero hay error menor: diagnóstico normal.

⚠ Nota: La caída de PI Vision en planta desaladora es automáticamente P1. No esperar confirmación adicional.
Paso 01P1
[PROTOCOLO P1] Activar incidente crítico

Notificar INMEDIATAMENTE: (1) ADC Pablo Piombi, (2) Supervisor de planta, (3) Equipo Pinnovartic. Abrir canal de comunicación dedicado (WhatsApp o Teams). Establecer comunicación de estado cada 15 minutos. No cerrar el canal hasta que el sistema sea restaurado y estable.

⚠ Nota: La notificación P1 no espera diagnóstico completo. Notificar primero, diagnosticar en paralelo.
Paso 02
Reiniciar el servidor PI Vision

Conectarse al servidor PI Vision (ccmvepivadm o equivalente MBL) por escritorio remoto. Reiniciar el servicio PI Vision Web API y PI AF Services. Si los servicios no responden: reiniciar el servidor completo. Verificar que los servicios suben correctamente después del reinicio.

⚠ Nota: El reinicio del servidor PI Vision tarda aproximadamente 5-10 minutos en estar completamente operativo. Informar este tiempo al equipo de planta.
Paso 03
Verificar restauración del reporte de planta

Acceder a PI Vision desde un equipo cliente. Verificar que el reporte de la planta (Desaladora, Planta MVE, etc.) muestra datos actualizados. Confirmar con el operador de planta que el reporte está normalizado.

Paso 04
Coordinación con Pinnovartic si persiste el problema

Si el reinicio no resuelve el problema o el sistema tiene comportamiento anormal: contactar a Pinnovartic para soporte especializado. Compartir los logs del servidor y la descripción detallada del síntoma.

⚠ Nota: Pinnovartic tiene acceso especializado a la plataforma PI. Para problemas que van más allá de reinicio del servicio, siempre involucrarlos.
Paso 05
Cerrar y documentar

Una vez restaurado: documentar el RCA, las acciones tomadas y el tiempo de restauración. Para incidentes P1: redactar informe post-incidente dentro de las 24 horas siguientes. Cerrar el ticket.

📡 Criterios de Escalamiento
  • Servidor no responde a reinicio → Reinicio físico coordinado con operaciones del data center (si aplica)
  • Problema en la plataforma PI (tags, integración SCADA) → Pinnovartic + soporte OSIsoft
