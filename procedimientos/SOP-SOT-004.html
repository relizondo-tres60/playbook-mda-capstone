🎯 Propósito del Procedimiento

Gestionar solicitudes de restauración de backup y recuperación de datos de usuarios o sistemas, ejecutando el proceso con las aprobaciones requeridas y verificando la integridad de los datos recuperados.

📋 Alcance

✓ Incluye

  • Recuperación de archivos borrados accidentalmente
  • Restauración de datos de usuario desde backup
  • Verificación de disponibilidad y estado de backups

✗ No incluye

  • Recuperación de datos de disco dañado sin backup previo (proceso de recuperación forense externo)
  • Backup de datos de usuario individual en OneDrive (responsabilidad del usuario — OneDrive hace sincronización automática)
✅ Requisitos Previos
  • Autorización del ADC para ejecutar la restauración
  • Sistema de backup con datos del período solicitado
  • Ventana de mantenimiento aprobada para restauración en producción
📄 Procedimiento Paso a Paso
Paso 01
Verificar la solicitud y autorización del ADC

Toda restauración de backup requiere autorización escrita del ADC. Verificar: ¿qué datos se necesitan restaurar? ¿de qué fecha/hora? ¿a qué destino? Sin autorización del ADC: no proceder.

Paso 02
Verificar disponibilidad del backup

Consultar el sistema de backup para confirmar que existe un punto de restauración del período solicitado. Verificar la integridad del backup (sin errores de lectura). Confirmar el tiempo estimado de restauración.

Paso 03
Restaurar en ambiente de prueba primero

Si es posible: restaurar primero en un ambiente de prueba para verificar que los datos son correctos e íntegros. Solo después restaurar en producción o entregar al usuario.

⚠ Nota: Este paso evita sobreescribir datos más nuevos con datos del backup. Siempre restaurar en modo de prueba primero.
Paso 04
Ejecutar restauración y verificar con el usuario

Ejecutar la restauración con la autorización del ADC y en la ventana de mantenimiento aprobada. Verificar con el usuario que los datos recuperados son los que necesitaba y están íntegros. Cerrar con 'Resolved'.

📡 Criterios de Escalamiento
  • Sin backup disponible para el período solicitado → Notificar al ADC; evaluar recuperación forense si los datos son críticos
  • Restauración que tarda más de lo esperado y afecta disponibilidad → Coordinar con ADC para decisión de continuar o posponer

📝 Notas operacionales:
OneDrive sincroniza automáticamente los documentos de los usuarios. La mayoría de las 'recuperaciones de datos' se resuelven verificando la papelera de OneDrive o el historial de versiones. Esto debe verificarse ANTES de ejecutar una restauración formal de backup.
