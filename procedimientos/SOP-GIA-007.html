🎯 Propósito del Procedimiento

Gestionar la autenticación multifactor (MFA) en Microsoft Entra ID para los tres flujos operacionales: configuración inicial, cambio de dispositivo de autenticación y bloqueo por incumplimiento de política.

📋 Alcance

✓ Incluye

  • Configuración inicial de MFA para nuevos usuarios
  • Reconfiguraci de MFA por cambio de dispositivo móvil
  • Bloqueo condicional por incumplimiento de registro MFA en plazo definido
  • Verificación de estado MFA ante problemas de acceso

✗ No incluye

  • Problemas de MFA para cuentas SAP (ver SOP-SAP-002)
  • Configuración de MFA para cuentas administrativas especiales (N3)
✅ Requisitos Previos
  • Acceso al portal Entra ID (portal.azure.com) o Microsoft Entra admin center
  • Teléfono móvil del usuario con Microsoft Authenticator disponible
📄 Procedimiento Paso a Paso
Paso 01
Identificar el flujo MFA aplicable

Determinar cuál de los tres flujos aplica: (A) Configuración inicial: usuario nuevo o que nunca configuró MFA. (B) Cambio de dispositivo: usuario con MFA previo pero cambió o perdió el teléfono. (C) Bloqueo por incumplimiento: usuario que no configuró MFA en el plazo y fue bloqueado condicionalmente. Seleccionar el flujo correspondiente y seguir los pasos de ese flujo.

⚠ Nota: El flujo correcto determina si se debe revocar el MFA anterior antes de configurar uno nuevo.
Paso 02A
[Flujo A] Configuración inicial — Guiar al usuario

Enviar al usuario el enlace de registro MFA: https://aka.ms/mfasetup. Guiar paso a paso: (1) Descargar Microsoft Authenticator en el móvil. (2) En el portal web, hacer clic en 'Add sign-in method' > Authenticator app. (3) Escanear el código QR con la app. (4) Verificar con un código de prueba. Confirmar que el usuario puede autenticarse exitosamente.

⚠ Nota: Método preferido: Microsoft Authenticator. Método alternativo aceptado: SMS al número personal registrado. No configurar correo alternativo como único método MFA.
Paso 02B
[Flujo B] Cambio de dispositivo — Revocar y re-registrar

Acceder a Entra ID > Users > buscar usuario > Authentication Methods. Eliminar el método MFA anterior (el dispositivo perdido o cambiado). Una vez removido, guiar al usuario con el mismo flujo A para registrar el nuevo dispositivo.

⚠ Nota: No re-registrar sin revocar el método anterior. El dispositivo antiguo podría ser usado por un tercero.
Paso 02C
[Flujo C] Bloqueo por incumplimiento — Aplicar o levantar bloqueo condicional

Si el usuario NO ha configurado MFA en el plazo definido: agregar la cuenta al grupo de seguridad de bloqueo condicional (GRP-MFA-Enforcement o equivalente). Notificar al usuario por correo personal o teléfono que su cuenta tiene acceso restringido hasta configurar MFA. Si el usuario ya configuró MFA: remover del grupo de bloqueo condicional y verificar que el acceso se restaura.

⚠ Nota: El grupo de bloqueo condicional debe estar pre-configurado en Entra ID Conditional Access. Si no existe, coordinar con N3 para su creación.
Paso 03
Verificar acceso post-configuración

Solicitar al usuario que cierre sesión y vuelva a iniciar. Verificar que el proceso de MFA aparece correctamente al iniciar sesión y que el usuario puede completarlo exitosamente. Si el acceso sigue fallando, verificar que no hay otras políticas condicionales activas.

Paso 04
Documentar y cerrar

Registrar en el ticket el flujo ejecutado y el resultado. Para flujo C, indicar si se aplicó o se levantó el bloqueo. Adjuntar captura del estado MFA del usuario en Entra ID. Cerrar con 'Resolved'.

📡 Criterios de Escalamiento
  • Si el portal de Entra ID no permite revocar MFA del usuario → N3 Server_Support
  • Si la política condicional no responde al agregar/remover del grupo → N3 (problema de Conditional Access)
  • Si el usuario no tiene dispositivo móvil disponible en zona sin señal (mina) → Coordinar con supervisor para acceso temporal

Los tres flujos son distintos. El error más común es no revocar el método anterior en el flujo B. Siempre verificar el estado actual del MFA del usuario en Entra ID antes de proceder.
